# GitHub Monetization Playbook

This repo is now wired for GitHub Sponsors via `.github/FUNDING.yml`.

Use this sequence to convert that setup into revenue.

## 1. Activate Sponsors (Account)

1. Open: `https://github.com/sponsors`
2. Select account: `question86`
3. Complete profile intro, bank setup, tax form, and 2FA requirement
4. Submit for approval

## 2. Publish Initial Tiers

Start with three monthly tiers:

1. `$5` Supporter
2. `$25` Builder
3. `$200` Team

Copy baseline text from `SPONSORSHIP.md`.

## 3. Add Sponsor Value

1. Create a private sponsor repo (for updates and early access)
2. Attach that repo to the `$25`+ tier in Sponsors settings
3. Post one meaningful sponsor update per week

## 4. Increase Conversion

1. Keep sponsor CTA near top of README
2. Add sponsor CTA to release notes
3. Add sponsor CTA to pinned profile repositories

## 5. Weekly Operating Loop

1. Publish one improvement with measurable value
2. Post one short "what shipped / what next" update
3. Ask users for sponsorship after issue resolution
4. Retire weak tiers and replace with clearer outcomes

## 6. Scale Beyond Donations

After sponsor traction:

1. Package a focused GitHub Action or App
2. Publish on GitHub Marketplace
3. Charge based on team size or usage
