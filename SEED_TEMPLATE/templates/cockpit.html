🔢 Token Usage Monitor (ESTIMATE) ℹ️ How it works

📚 Token Budget Explained
⚠️ Token Tracking Limitation:
VS Code/GitHub Copilot does not expose real-time token usage via API. The values shown are manual estimates that you can update based on the conversation summary info that appears during long sessions.

What is a token?
A token is a unit of text processing. On average, 1 token ≈ 4 characters or ≈ 0.75 words. Example: "Hello world!" = ~3 tokens

1 Million Token Budget:
Your session has a total budget of 1,000,000 tokens shared between:
• Input tokens: Your prompts + context (files read, code examined)
• Output tokens: AI responses (code generated, explanations)

Token Consumption:
• Reading a file: ~500-2000 tokens (depends on file size)
• Simple question: ~200-500 tokens (input + output)
• Complex task: ~15,000-30,000 tokens (with file edits)
• Creating a report: ~8,000-12,000 tokens

Current Display:
• Current Session: Estimated tokens used so far in this chat
• Remaining: Tokens left before budget exhausted
• Est. Per Task: Typical token usage by task type

💡 Pro Tip: The loop system's amnesia-by-design resets token budget each loop, preventing budget exhaustion across long projects.

🎯 Loop Closure Timing Guidance:
• 0-60% (0-600K tokens): GREEN ZONE - Optimal for continuing work. Reports are concise but complete.
• 60-85% (600K-850K tokens): YELLOW ZONE - Consider finalizing soon. Reports may become verbose but thorough.
• 85-100% (850K-1M tokens): RED ZONE - Finalize loop ASAP. Risk of incomplete tasks or missing context.

Recommended: Finalize loops in the 70-85% range for balance between productivity and quality.
